# 用户画像:
中小学班主任及语文教师，负责班级早读管理，面临学生开口率低、朗读音量不足、课堂氛围沉闷等问题，传统口头督促方式效果有限且消耗大量课堂精力。
# 场景:
中小学早读课堂。教师发起朗读任务后，学生集体朗读课文，但存在声音参差不、部分学生默读或走神的情况教师需要实时手段维持朗读音量与课堂专注度。
# 需求:
一款游戏化分贝监测工具，支持教师设定目标音量阈值，将学生朗读音量实时转化为屏幕上的像素动物动态反馈-音量达标即触发动物出现，持续达标则动物不断繁衍增多，以即时可视化激励替代教师反复督促，低成本提升学生早读参与度与朗读音量，营造积极投入的晨读氛围。

# 设计方案

## 一、核心功能模块详细设计

### 1.1 教师控制面板

| 功能点 | 描述 | 交互规则 |
|--------|------|---------|
| 阈值设定 | 滑块控件，范围 40–90 dB，步长 1 dB | 默认值 60 dB；拖动时实时预览；松开后持久化到 localStorage |
| 早读任务控制 | 三个按钮：开始 / 暂停 / 结束 | 就绪态仅"开始"可点；进行中"暂停""结束"可点；暂停态"继续""结束"可点 |
| 朗读内容选择 | 下拉或标签选择（课文/古诗词/自由朗读） | 可选字段，用于后续统计分类 |
| 历史记录入口 | 图标按钮，打开历史达标记录面板 | 展示最近 30 条记录，按日期倒序 |

### 1.2 音频采集与分贝计算

```
MediaStream ──→ AudioContext ──→ AnalyserNode ──→ 时域数据 (Uint8Array)
                                                       │
                                                       ▼
                                              RMS 计算 (均方根)
                                                       │
                                                       ▼
                                              dB = 20 × log₁₀(RMS / ref)
                                                       │
                                                       ▼
                                              阈值比较 → 事件下发
```

**技术选型**：Web Audio API（`navigator.mediaDevices.getUserMedia`）

**关键参数**：
- 采样率：44.1 kHz（浏览器默认）
- FFT size：2048（平衡精度与性能）
- 平滑系数：0.8（避免数值跳变）
- 计算频率：每 100ms 计算一次（10 Hz 刷新率）

**分贝映射表**（供教师参考设定阈值）：

| 场景 | 参考 dB | 建议阈值 |
|------|--------|---------|
| 安静教室 | ~30 dB | — |
| 小声朗读 | 45–55 dB | — |
| 正常朗读 | 55–70 dB | 60 dB（默认） |
| 大声齐读 | 70–85 dB | 75 dB（挑战模式） |

### 1.3 游戏化激励引擎（核心）

#### 1.3.1 像素动物系统

| 属性 | 说明 |
|------|------|
| 动物种类 | 猫、狗、兔子、小鸡、熊猫（5 种，后续可扩展） |
| 像素规格 | 16×16 px 精灵，4 帧行走动画 |
| 画布尺寸 | 响应式，最大 960×540 px |
| 动物上限 | 画面同时存在 ≤ 200 只（防止性能劣化） |
| 颜色方案 | 每种动物 3–4 种配色变体，增加视觉丰富度 |

#### 1.3.2 出生/繁衍规则

```
当前周期 dB ≥ 阈值 ──────────────────→ 出生 1 只新动物（随机种类+配色）
                                          │
连续达标 ≥ 5 秒 ──────────────────────→ 繁衍加速：每 2 秒出生 1 只
连续达标 ≥ 15 秒 ─────────────────────→ Combo 触发：一次出生 3 只 + 特效
连续达标 ≥ 30 秒 ─────────────────────→ 暴击触发：一次出生 8 只 + 满屏特效
```

#### 1.3.3 衰减/消亡规则

```
dB < 阈值 且 持续 < 3 秒 ──────────────→ 停止生成新动物（已有动物保持）
dB < 阈值 且 持续 3–10 秒 ─────────────→ 动物每秒消失 1 只（淡出动画）
dB < 阈值 且 持续 > 10 秒 ─────────────→ 动物加速消失 + 屏幕边缘变红提示
```

#### 1.3.4 Combo 连击系统

| Combo 等级 | 条件 | 视觉反馈 |
|-----------|------|---------|
| 🔥 火热 | 连续达标 15s | 屏幕边框金色脉动 |
| ⚡ 暴风 | 连续达标 30s | 彩色粒子飘落 + 动物奔跑加速 |
| 🌈 传说 | 连续达标 60s | 出现稀有金色动物 + 全屏烟花粒子 |

## 二、状态机设计

### 2.1 早读任务状态机

```
                    ┌─────────┐
                    │  就绪    │ ← 页面加载 / 上次任务结束
                    │  IDLE   │
                    └────┬────┘
                         │ 教师点击"开始"
                         ▼
                    ┌─────────┐
            ┌──────→│  进行中  │
            │       │ RUNNING │
            │       └────┬────┘
            │            │
            │  教师点击    │ 教师点击
            │  "暂停"     │ "结束"
            │            │
            │       ┌────┴────┐
            │       ▼         ▼
            │  ┌─────────┐  ┌─────────┐
            │  │  暂停    │  │  已结束   │
            │  │ PAUSED  │  │ FINISHED │
            │  └────┬────┘  └─────────┘
            │       │
            │       │ 教师点击"继续"
            └───────┘
```

**状态说明**：
- **IDLE**：麦克风未激活，画布空闲，可配置参数
- **RUNNING**：麦克风采集音频，实时计算 dB，动物动态刷新
- **PAUSED**：麦克风暂停，动物冻结，保留当前画面
- **FINISHED**：停止采集，汇总数据，展示结果面板

## 三、数据模型设计

### 3.1 早读会话 (Session)

```typescript
interface Session {
  id: string;                    // UUID，唯一标识
  startTime: number;             // 开始时间戳 (ms)
  endTime: number | null;        // 结束时间戳 (ms)，进行中为 null
  status: 'idle' | 'running' | 'paused' | 'finished';
  config: SessionConfig;         // 本次会话配置
  stats: SessionStats;           // 实时统计数据
}

interface SessionConfig {
  thresholdDb: number;           // 目标分贝阈值 (默认 60)
  readingContent: string;        // 朗读内容名称
  animalTypes: string[];         // 启用的动物类型 (默认全部)
}
```

### 3.2 实时统计 (SessionStats)

```typescript
interface SessionStats {
  totalDuration: number;         // 总时长 (秒)
 达标Duration: number;          // 达标累计时长 (秒)
 达标Rate: number;              // 达标率 (0–1)
  currentDb: number;             // 当前实时分贝值
  peakDb: number;                // 本次最高分贝
  avgDb: number;                 // 本次平均分贝
  animalCount: number;           // 当前画面动物数量
  totalAnimalsSpawned: number;   // 累计出生动物总数
  comboLevel: 0 | 1 | 2 | 3;   // 当前 Combo 等级
  comboStreak: number;           // 当前连击秒数
}
```

### 3.3 动物实例 (Animal)

```typescript
interface Animal {
  id: string;                    // 唯一标识
  type: AnimalType;              // 动物种类
  colorVariant: number;          // 配色变体 (0–3)
  x: number;                     // 画布横坐标 (px)
  y: number;                     // 画布纵坐标 (px)
  state: 'spawn' | 'idle' | 'walk' | 'breed' | 'fade';
  frameIndex: number;            // 当前动画帧 (0–3)
  direction: 1 | -1;            // 朝向 (右/左)
  speed: number;                 // 移动速度 (px/frame)
  spawnTime: number;             // 出生时间戳
}

type AnimalType = 'cat' | 'dog' | 'rabbit' | 'chick' | 'panda';
```

### 3.4 历史记录 (HistoryRecord)

```typescript
interface HistoryRecord {
  id: string;
  date: string;                  // 日期 YYYY-MM-DD
  sessionStats: SessionStats;    // 最终统计数据
  config: SessionConfig;
  createdAt: number;             // 记录创建时间戳
}
```

### 3.5 存储方案

| 数据 | 存储位置 | Key | 说明 |
|------|---------|-----|------|
| 当前会话 | 内存 (React state) | — | 页面关闭即丢失 |
| 阈值偏好 | localStorage | `zaodu:config` | 下次打开自动恢复 |
| 历史记录 | localStorage | `zaodu:history` | 最多保留 30 条，FIFO |
| 动物精灵 | 静态资源 / Base64 | — | 打包到前端项目中 |

## 五、关键算法伪代码

### 5.1 实时分贝计算

```javascript
// 每个计算周期 (100ms) 执行一次
function calculateDecibel(analyserNode) {
  const buffer = new Uint8Array(analyserNode.fftSize);
  analyserNode.getByteTimeDomainData(buffer);

  // 1. 计算 RMS (均方根)
  let sumSquares = 0;
  for (let i = 0; i < buffer.length; i++) {
    const normalized = (buffer[i] - 128) / 128;  // 归一化到 [-1, 1]
    sumSquares += normalized * normalized;
  }
  const rms = Math.sqrt(sumSquares / buffer.length);

  // 2. RMS → dB (以 20μPa 为参考，归一化后简化计算)
  const db = 20 * Math.log10(rms / 0.00002);  // 参考值 = 0 dB SPL 对应

  // 3. 平滑处理 (指数移动平均)
  const smoothedDb = previousDb * SMOOTHING + db * (1 - SMOOTHING);
  // SMOOTHING = 0.8

  return Math.round(smoothedDb);
}
```

### 5.2 动物繁衍判定

```javascript
function updateAnimalPopulation(currentDb, thresholdDb, comboStreak, animals) {
  const MAX_ANIMALS = 200;

  // 1. 达标判断
  if (currentDb >= thresholdDb) {
    comboStreak += 0.1;  // 累计连击秒数 (100ms 周期)

    // 2. 繁衍速率
    let spawnInterval;
    if (comboStreak >= 60)      spawnInterval = 0.5;   // 传说: 每0.5s
    else if (comboStreak >= 30) spawnInterval = 1.0;   // 暴风: 每1s
    else if (comboStreak >= 15) spawnInterval = 1.5;   // 火热: 每1.5s
    else if (comboStreak >= 5)  spawnInterval = 2.0;   // 加速: 每2s
    else                        spawnInterval = 3.0;   // 常规: 每3s

    // 3. 批量出生
    let spawnCount = 1;
    if (comboStreak >= 60)      spawnCount = 8;
    else if (comboStreak >= 30) spawnCount = 3;

    if (animals.length < MAX_ANIMALS && shouldSpawn(spawnInterval)) {
      spawnAnimals(spawnCount, animals);
    }

  } else {
    // 4. 未达标处理
    comboStreak = Math.max(0, comboStreak - 0.2);

    // 5. 衰减逻辑
    if (comboStreak === 0) {
      const decaySeconds = timeBelowThreshold;
      if (decaySeconds > 10) {
        removeAnimals(3, animals);  // 加速消失
      } else if (decaySeconds > 3) {
        removeAnimals(1, animals);  // 缓慢消失
      }
    }
  }

  return comboStreak;
}
```
